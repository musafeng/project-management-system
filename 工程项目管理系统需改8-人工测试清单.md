# 工程项目管理系统需改8-人工测试清单

适用范围：仅针对 `工程项目管理系统需改8.docx` 提到的问题，以及本轮同步补上的同类型兼容问题。  
执行原则：不发散，不做大范围探索，只验证本轮修正是否可作为上线前最后一次收口。

## 测试说明

- 建议先用日常业务账号验证提单、提交审批、导出。
- 审批流相关场景，需再用实际审批人账号完成流转。
- 每个模块至少测 1 条成功路径；如发现异常，记录页面、操作步骤、提示文案、是否带附件。

## 一、项目合同变更

### 1. 新增合同变更

- [ ] 进入 `项目合同变更` 页面。
- [ ] 新增一条记录，填写：项目合同、变更日期、增项金额、备注、附件。
- [ ] 点击 `确定` 后保存成功，不出现英文报错。
- [ ] 列表中新增记录可正常显示。
- [ ] 新增后状态显示为 `待提交`，而不是异常状态。

### 2. 提交审批

- [ ] 在刚新增的合同变更记录上点击 `提交审批`。
- [ ] 页面提示为中文，不出现英文报错。
- [ ] 提交成功后状态变为 `待审批`。
- [ ] 审批中心可看到对应待办/我发起的记录。

### 3. 审批通过后联动

- [ ] 用实际审批流账号完成该条合同变更审批。
- [ ] 审批最终通过后，回到 `项目合同管理` 查看原合同。
- [ ] 合同金额更新为变更后的金额。
- [ ] `变更后金额 / 应收金额 / 未收金额` 联动正确。

验收口径：
- 新增成功。
- 提交审批成功。
- 合同金额在“最终审批通过后”更新，不要求在“仅提交审批”时更新。

### 3. 分包付款

## 四、附件上传

### 1. 单附件上传

- [ ] 任意一个含附件的单据页面，上传 1 个 PDF。
- [ ] 上传成功，页面显示附件链接。
- [ ] 保存后再次打开记录，附件仍可查看。

### 2. 多 PDF 上传

- [ ] 在支持附件的表单中一次选择多个 PDF 文件。
- [ ] 上传过程中不出现 `操作失败、请稍候再试`。
- [ ] 所有文件都成功显示在附件列表中。
- [ ] 保存后再次打开记录，多个附件仍能回显。

### 3. 明细附件

- [ ] 在费用报销类页面的明细行中上传附件。
- [ ] 保存后再次打开记录，明细附件仍可查看。

重点说明：
- 这一项是本轮最需要人工确认的场景。
- 若失败，请务必记录：具体页面、上传文件数量、文件大小、文件名后缀、失败提示原文。

## 五、按项目导出

### 1. 页面跳转

- [ ] 从业务页面点击 `导出数据`。
- [ ] 能正常跳转到 `数据下载中心`。

### 2. 项目筛选预览

- [ ] 在 `数据下载中心` 选择对应业务模块。
- [ ] 选择一个明确有数据的项目。
- [ ] 点击 `查询预览`。
- [ ] 页面能查询出数据，不出现异常或空白错误。

### 3. 下载 CSV

- [ ] 在已有预览数据的前提下点击 `下载 CSV`。
- [ ] 下载成功，文件可打开。
- [ ] CSV 内容与筛选条件一致。

建议至少补测以下导出入口：
- [ ] 项目合同
- [ ] 合同收款
- [ ] 其他收款
- [ ] 其他付款
- [ ] 管理费用报销
- [ ] 销售费用报销
- [ ] 备用金申请

验收口径：
- 按项目筛选时不再出现“跳转后无数据但原页面实际有数据”的情况。
